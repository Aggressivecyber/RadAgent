# RadAgent 未来展望与国奖冲刺优化路线

> 目标：将 RadAgent 从“能自动跑 Geant4 的工具”升级为“面向航天辐照防护设计的可信智能仿真平台”，形成可展示、可验证、可复现、可扩展的完整科研工程闭环。

## 1. 项目当前定位

RadAgent 当前已经具备一个较完整的自动化链路：

1. 用户用自然语言描述航天辐照仿真需求。
2. 系统解析轨道、粒子、材料、屏蔽结构等关键参数。
3. 调研子图生成仿真计划，并进入人工确认。
4. 模板引擎生成 Geant4 工程源码。
5. 自动编译、运行、采集 CSV 结果。
6. 分析阶段生成几何示意图、能量热力图、能谱图。
7. 报告节点生成仿真报告并进入人工审核。

这已经不是单点脚本，而是一个具备“规划-执行-分析-报告”闭环的工程智能体。

但是，如果目标是冲击国奖，当前定位还不够锋利。评审不会只看“能不能跑”，更会看：

- 是否解决了真实科研/工程痛点。
- 是否有技术创新，而不是简单调用大模型。
- 是否有可量化结果，而不是主观演示。
- 是否具有可靠性、可复现性和工程可扩展性。
- 是否有展示冲击力，能让非本方向评委快速理解价值。

因此，后续应把项目定位升级为：

> RadAgent 是一个面向航天器辐照防护设计的可信智能仿真平台，融合大模型规划、领域知识库、Geant4 蒙特卡洛仿真、自动化质量门禁、设计优化与可复现实验报告生成，实现从自然语言需求到可验证仿真结论的一站式闭环。

## 2. 国奖视角下的核心卖点

### 2.1 从“自动化”升级为“智能化”

普通自动化系统是固定流程：输入参数，生成模板，运行仿真。

RadAgent 应强调的是智能化：

- 能理解模糊需求。
- 能补全缺失参数。
- 能基于轨道和任务场景推断粒子环境。
- 能根据材料与结构约束生成合理仿真方案。
- 能识别异常仿真结果。
- 能给出设计建议，而不只是输出图表。

答辩表达：

> 本项目不是把 Geant4 命令行封装成按钮，而是构建了一个具备领域知识、仿真执行、异常诊断和报告生成能力的科研智能体。

### 2.2 从“仿真工具”升级为“设计平台”

仿真工具回答的问题是：

> 这个结构的剂量是多少？

设计平台回答的问题是：

> 在质量、厚度、材料和辐照约束下，什么结构更优？

这是质的区别。国奖评审更容易认可后者，因为它面向真实工程决策。

后续应加入：

- 多材料组合搜索。
- 厚度参数扫描。
- 敏感体积剂量约束。
- 质量/屏蔽效果权衡。
- Pareto 前沿可视化。
- 自动推荐最优结构。

答辩表达：

> 系统不仅能复现工程师给定的仿真方案，还能在约束条件下自动探索屏蔽结构设计空间，为航天器防护方案提供决策支持。

### 2.3 从“结果生成”升级为“可信验证”

大模型系统最容易被质疑的是可靠性。RadAgent 要主动把可靠性设计成项目亮点。

可信性可以拆成四层：

1. 输入可信：参数范围、材料名称、单位转换、任务场景合理性。
2. 过程可信：生成源码可追溯、编译日志可检查、运行宏可复现。
3. 结果可信：能量沉积、剂量、峰值层、穿透情况有异常检测。
4. 报告可信：结论有数据来源，图表与原始 CSV 对应，风险有标注。

答辩表达：

> 针对大模型生成式系统容易出现幻觉和不可复现的问题，我们设计了多阶段质量门禁与实验追踪机制，使智能体输出从“看起来合理”变成“可检查、可复现、可审计”。

## 3. 当前短板

### 3.1 缺少硬指标支撑

目前项目能展示流程，但缺少可量化评估。没有指标，评审会觉得只是 demo。

需要补：

- 人工建模时间 vs RadAgent 建模时间。
- 人工报告时间 vs RadAgent 报告时间。
- 生成 Geant4 工程的一次成功率。
- 编译通过率。
- 仿真结果与基准案例的相对误差。
- 复杂需求下的任务完成率。

### 3.2 缺少标准案例对标

Geant4 是严肃科学计算工具。要证明系统可信，应至少选几个标准对标案例：

- 单层铝屏蔽质子入射。
- 多层 Al/PE/Si 屏蔽。
- 电子在低 Z 材料中的能量沉积。
- gamma 或 neutron 的基础穿透案例。

每个案例应保留：

- 输入需求。
- 人工基准模型。
- RadAgent 自动模型。
- 剂量/能量沉积对比。
- 误差解释。

### 3.3 报告展示仍偏“自动生成文本”

Markdown 报告有用，但答辩现场冲击力不够。

应补一个 HTML dashboard：

- 顶部显示任务摘要和风险等级。
- 左侧显示仿真参数。
- 中间显示几何结构。
- 右侧显示关键指标卡片。
- 下方显示热力图、深度剂量、能谱和结论。

评委不一定懂 Geant4，但能看懂一个专业 dashboard。

### 3.4 设计优化能力不足

当前系统偏执行型。想冲奖，必须补“决策型”能力。

最小可行版本：

- 输入剂量阈值和质量约束。
- 自动扫描 10-30 个候选结构。
- 生成候选排名。
- 输出推荐方案。

高级版本：

- 使用贝叶斯优化或遗传算法。
- 多目标优化：敏感体积剂量最小、质量最小、厚度最小、成本最低。
- 输出 Pareto 曲线。

### 3.5 领域知识沉淀不足

系统需要显式体现“航天辐照领域知识”，否则容易被认为只是通用 Agent 套壳。

应沉淀：

- 材料知识库。
- 轨道环境知识库。
- 粒子类型与典型能量范围。
- 常见航天器敏感部件。
- 屏蔽设计经验规则。
- Geant4 物理列表选择规则。

## 4. 深度优化方向

## 4.1 可信仿真质量门禁

建议把门禁设计成项目核心创新之一。

### 输入门禁

检查项：

- 厚度是否为正。
- 密度是否与材料一致。
- 材料是否能映射到 Geant4 NIST 材料。
- 粒子能量是否在合理范围。
- 事件数是否足够支撑统计稳定性。
- 轨道高度、倾角是否合理。
- 敏感体积是否存在于几何层中。

输出形式：

```json
{
  "status": "pass | warning | fail",
  "issues": [
    {
      "field": "particle.energy_MeV",
      "level": "warning",
      "message": "10 MeV proton may stop in outer aluminum layer; sensitivity volume dose may be near zero."
    }
  ]
}
```

### 编译门禁

检查项：

- CMake 是否成功。
- Geant4 头文件是否缺失。
- 材料名称是否非法。
- 模板占位符是否未渲染。
- 生成的 C++ 是否存在明显语法错误。

优化方向：

- 编译失败后自动读取错误日志。
- 归因到模板、材料、单位或 API 使用问题。
- 尝试自动修复。
- 记录修复轨迹。

### 仿真结果门禁

检查项：

- CSV 是否存在。
- 步进数是否为 0。
- 事件数是否符合预期。
- 总能量沉积是否为 0。
- 敏感体积是否被命中。
- 剂量是否异常大或异常小。
- 峰值层是否符合物理直觉。
- 深度剂量曲线是否存在非物理突变。

### 报告门禁

检查项：

- 报告是否包含配置、结果、图表、异常、结论。
- 图表路径是否存在。
- 报告中的数值是否与原始结果一致。
- 结论是否过度推断。
- 是否标明局限性。

## 4.2 设计优化模块

建议新增一个 `optimizer` 阶段。

当前流程：

```text
research -> parameterize -> build_and_run -> analyze -> report
```

增强流程：

```text
research -> candidate_generation -> batch_simulation -> optimization_analysis -> report
```

### 候选结构生成

输入：

- 目标剂量阈值。
- 最大厚度。
- 最大面密度。
- 候选材料集合。
- 任务粒子环境。

输出：

- 多个候选层状结构。

示例：

```json
[
  {
    "name": "Al-PE-Si baseline",
    "layers": [
      {"material": "G4_Al", "thickness_mm": 10},
      {"material": "G4_POLYETHYLENE", "thickness_mm": 5},
      {"material": "G4_Si", "thickness_mm": 2}
    ]
  }
]
```

### 目标函数

可以先做简单形式：

```text
score = w1 * normalized_sensitive_dose
      + w2 * normalized_areal_density
      + w3 * normalized_total_thickness
      + penalty_if_dose_exceeds_threshold
```

推荐输出：

- 最低剂量方案。
- 最轻方案。
- 综合最优方案。
- Pareto 前沿方案。

### 可视化

应新增：

- 剂量-质量散点图。
- 厚度-剂量曲线。
- Pareto 前沿图。
- 候选结构排名表。

这会显著提升项目层次。

## 4.3 专业知识库

建议建立 `knowledge/` 目录，至少包含：

```text
knowledge/
  materials.yaml
  orbit_environments.yaml
  particle_spectra.yaml
  physics_lists.yaml
  design_rules.yaml
  validation_cases.yaml
```

### 材料库

字段建议：

```yaml
- name_zh: 铝
  name_en: Aluminum
  geant4_name: G4_Al
  density_g_cm3: 2.699
  role: shield
  notes: 结构材料，常用于航天器外壳，对低能质子有较好阻挡能力。
```

### 轨道环境库

字段建议：

```yaml
- orbit: LEO
  altitude_range_km: [160, 2000]
  dominant_particles: [proton, electron]
  typical_energy_MeV:
    proton: [1, 400]
    electron: [0.1, 7]
```

### 设计规则库

示例规则：

- 低能质子屏蔽可优先考虑铝或聚乙烯组合。
- 电子屏蔽需注意轫致辐射，高 Z 材料不一定总是最优。
- 敏感体积应作为单独 scoring volume 输出剂量。
- 事件数不足时需要标记统计不确定性。

## 4.4 不确定性与统计可信度

Monte Carlo 仿真天然有统计误差。当前报告如果只给一个剂量值，会显得不够科研化。

建议补：

- 多 seed 重复仿真。
- 均值、标准差、置信区间。
- 事件数收敛曲线。
- 相对误差估计。

输出形式：

```text
Sensitive volume dose = 2.4e-6 Gy ± 0.2e-6 Gy (95% CI)
```

评审表达：

> 我们不仅输出单次仿真结果，还通过多随机种子重复实验估计统计不确定性，使结果具有 Monte Carlo 置信度。

## 4.5 可复现实验包

建议每次运行自动导出：

```text
experiment_package/
  input_request.txt
  sim_plan.json
  generated_geant4/
  build.log
  run.log
  raw_csv/
  analysis_data.json
  figures/
  report.md
  report.html
  manifest.json
```

`manifest.json` 应包含：

- 时间戳。
- git commit。
- Python 版本。
- Geant4 版本。
- 运行命令。
- 随机种子。
- 文件 hash。

这会把项目从“能跑”提升为“可审计”。

## 4.6 HTML Dashboard 报告

建议新增 `report_html.py`。

页面结构：

1. Header：任务名称、风险等级、运行状态。
2. Summary cards：
   - 总剂量。
   - 敏感体积剂量。
   - 峰值层。
   - 是否穿透。
   - 事件数。
3. Geometry panel：层状结构示意图。
4. Figures：
   - 2D energy heatmap。
   - depth-dose profile。
   - layer spectra。
5. Findings：
   - 主要结论。
   - 异常说明。
   - 设计建议。
6. Reproducibility：
   - 源码路径。
   - 原始数据路径。
   - 参数 JSON。

HTML 报告对答辩非常重要，因为它能让评委快速建立“这是一个完整系统”的印象。

## 4.7 3D 几何预览

可以增加一个轻量 Three.js 预览，而不一定要接入完整 Geant4 可视化。

功能：

- 多层屏蔽结构 3D 展示。
- 入射粒子箭头。
- 敏感体积高亮。
- 鼠标悬停显示材料、厚度、密度。
- 支持导出截图。

展示价值很高，尤其适合答辩视频。

## 4.8 自动修复能力

这是最像“智能体”的能力。

典型流程：

1. 生成 Geant4 工程。
2. 编译失败。
3. 读取错误日志。
4. 分类错误：
   - 材料名称错误。
   - C++ 模板错误。
   - Geant4 API 使用错误。
   - CMake 配置错误。
5. 自动修改。
6. 重新编译。
7. 保存修复记录。

建议指标：

- 常见错误自动修复成功率。
- 平均修复轮数。
- 修复前后编译成功率提升。

输入类型：

- 明确输入。
- 缺少材料。
- 缺少粒子能量。
- 中英文混合输入。
- 不合理参数输入。
- 多层复杂结构输入。

- 参数解析成功率。

实验设计：

- 固定目标：敏感体积剂量低于阈值。
- 固定约束：总厚度不超过某值。
- 比较 baseline 和 RadAgent 优化结果。

指标：

- 剂量降低比例。
- 质量降低比例。
- 候选搜索次数。
- 是否找到满足约束方案。

## 7. 国奖答辩叙事

### 7.1 痛点

航天器在轨运行面临复杂辐射环境。传统 Geant4 辐照仿真存在：

- 建模门槛高。
- 参数配置复杂。
- 调试成本高。
- 数据分析耗时。
- 报告整理繁琐。
- 非专家难以快速开展方案评估。

### 7.2 方案

RadAgent 将大模型智能规划、领域知识约束和 Geant4 蒙特卡洛仿真结合，构建从自然语言需求到可复现实验报告的自动化闭环。

### 7.3 创新点

建议归纳为四点：

1. 面向航天辐照场景的自然语言仿真任务规划。
2. 结合领域知识库与模板引擎的 Geant4 自动建模。
3. 多阶段可信门禁与异常诊断机制。
4. 面向屏蔽结构设计的自动优化与可视化报告生成。

### 7.4 成果

建议准备：

- 一个 2 分钟演示视频。
- 一个 dashboard 报告截图。
- 一张系统架构图。
- 一张 benchmark 表。
- 一张优化前后对比图。
- 一个可复现实验包。

### 7.5 结论

答辩收束语：

> RadAgent 降低了航天辐照仿真的使用门槛，提高了建模和报告效率，并通过可信门禁和可复现实验包增强了仿真结果的可靠性。未来可扩展到卫星电子器件防护设计、深空探测任务评估和多物理场仿真智能体平台。

## 8. 三阶段实施路线

### 阶段一：冲刺展示能力

目标：让项目“看起来完整、讲得清楚、演示稳定”。

任务：

- 完善三张分析图。
- 生成 HTML dashboard。
- 增加实验包导出。
- 选 3 个标准案例。
- 做效率对比表。

预计收益：

- 答辩展示质量显著提升。
- 评委能快速理解项目价值。

### 阶段二：增强可信性

目标：让项目“不只是 demo，而是可靠系统”。

任务：

- 输入门禁。
- 结果异常检测。
- 报告一致性检查。
- 多 seed 不确定性估计。
- 编译失败自动诊断。

预计收益：

- 抵御“AI 幻觉”和“结果可信吗”的质疑。
- 项目技术深度提升。

### 阶段三：形成设计优化平台

目标：让项目从“自动仿真”升级为“智能设计”。

任务：

- 材料库。
- 候选方案生成。
- 批量仿真。
- 多目标优化。
- Pareto 可视化。

预计收益：

- 项目创新性显著增强。
- 更容易形成论文、专利或竞赛成果。

## 9. 优先级建议

如果时间有限，建议按这个顺序：

1. HTML dashboard 报告。
2. Benchmark 对比实验。
3. 可复现实验包。
4. 输入/结果质量门禁。
5. 方案优化模块。
6. 知识库结构化。
7. 多 seed 不确定性。
8. 3D 几何预览。
9. 自动修复。

最小国奖冲刺组合：

```text
完整闭环 + 专业图表 + dashboard + benchmark + 可复现包 + 一个优化案例
```

这个组合比单纯加很多功能更有效。

## 10. 可量化目标

建议设定如下目标，后续用实验填真实数据：

- 自然语言需求解析成功率 >= 90%。
- Geant4 工程一次生成成功率 >= 85%。
- 编译成功率 >= 90%。
- 标准案例剂量相对误差 <= 10%。
- 报告生成时间降低 >= 90%。
- 建模总耗时降低 >= 80%。
- 异常输入拦截率 >= 90%。
- 优化案例中敏感体积剂量降低 >= 30%。

这些指标不一定一开始都达到，但需要有目标体系。

## 11. 论文与专利方向

### 论文方向

可写题目：

1. 面向航天辐照仿真的大模型智能体自动建模方法。
2. 基于 Geant4 与多阶段质量门禁的可信仿真智能体。
3. 面向航天器屏蔽结构设计的自然语言驱动蒙特卡洛仿真平台。

### 专利方向

可能的专利点：

1. 一种自然语言驱动的航天辐照仿真自动建模方法。
2. 一种面向蒙特卡洛仿真的多阶段质量门禁与异常诊断方法。
3. 一种基于智能体的航天器屏蔽结构自动优化方法。
4. 一种仿真实验自动追踪与可复现报告生成方法。

## 12. 最终愿景

RadAgent 的长期愿景不应局限于当前模板。

未来可以扩展为：

- 航天辐照仿真平台。
- 多物理场智能仿真平台。
- 科研自动化智能体框架。
- 面向工程设计的 AI + Simulation 工作流系统。

最终目标：

> 让非 Geant4 专家也能通过自然语言快速完成可信的航天辐照仿真，并让专业工程师能够用智能体批量探索设计空间、优化防护方案、生成可复现科研报告。
